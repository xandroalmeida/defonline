O que foi feito (EPIC‑000 · Fundação técnica)

Toda plataforma sustentável precisa de fundação invisível antes da primeira tela: ambientes, pipelines, observabilidade, padrões de qualidade e segurança. O EPIC‑000 entrega exatamente isso — sem ele, qualquer feature de produto seria construída em terreno instável.

Entregas concretas e auditáveis

  • Plataforma pública em homologação — primeira página da DEFOnline acessível com TLS válido, identificador de requisição rastreável fim‑a‑fim e endpoints técnicos de saúde (/health, /ready).
  • Esteira de entrega automatizada — cada candidata a release (tag rc.N) dispara deploy automático em homologação em menos de 10 minutos, sem intervenção manual. Produção segue o mesmo padrão com aprovação humana de 1 clique.
  • Gate de qualidade ativo — código abaixo de 80% de cobertura é bloqueado antes mesmo de chegar ao servidor; código que tocar regra de negócio (app/Domain) será bloqueado abaixo de 98% automaticamente quando essa pasta nascer no EPIC‑001.
  • Observabilidade básica — logs estruturados em JSON, métricas de requisição e de jobs persistidas, identificadores únicos por requisição, mascaramento de dados pessoais (LGPD) em todos os logs e eventos.
  • 6 decisões arquiteturais formalizadas, aceitas e indexadas (stack PHP/Laravel, topologia monolítica modular, persistência PostgreSQL multi‑tenant, infraestrutura como código via Ansible/Caddy, CI/CD GitHub Actions, observabilidade Postgres‑first). Cada decisão tem alternativas analisadas, custo orientativo e plano de evolução.
  • Validação independente — um agente Validador (papel separado de quem implementou) percorreu uma bateria de 25+ itens e produziu relatório auditável. Primeiro passe encontrou 1 inconsistência não‑bloqueante (gate de cobertura prescrito mas ainda não cabeado); correção entregue no mesmo dia; segundo passe aprovado.

Achado honesto da validação

O exercício de ativar o gate de cobertura revelou que a cobertura real do código era 57,7%, não os ≥80% que o time vinha declarando “de cabeça”. O time escreveu os testes faltantes — sem alterar lógica do produto — e a cobertura subiu para 92,4%. Esse tipo de descoberta é exatamente o motivo de existir uma camada de validação independente: declarar uma métrica é diferente de medi‑la. A partir deste ponto, qualquer regressão é bloqueada automaticamente pelo gate, em dois momentos (antes do código sair da máquina do dev e antes do pull‑request fundir na base).

Pontos para conversa com o board

Decisão recomendada nos próximos 5 dias

Disparar a revisão jurídica do Termo de Adesão e tratamento LGPD. É a única dependência externa do EPIC‑001 (cadastro). Se essa frente não andar em paralelo, o EPIC‑001 chega ao fim sem condições legais de aceitar usuários reais. Recomendação operacional: iniciar até 27/05/2026.

Conversa estratégica (sem urgência hoje)

Modelo de monetização do beta. A onda 1 fecha sem cobrança por desenho — o objetivo é validar valor, não receita. A decisão sobre trial gratuito de 7 dias vs. beta aberto indefinido precisa estar tomada antes da abertura da WAVE 2026‑02 (comercialização). Material de apoio: PDR‑001.

Apêndice — para leitores técnicos

Detalhes operacionais e rastreabilidade. Pode ser ignorado por leitores não‑técnicos.

  • Stack: Laravel 13 + Livewire 4 + PostgreSQL 18 + Pest 4 + Dusk 8 (PHP 8.5). Container único Docker para web/worker/scheduler. Reverse proxy Caddy 2. Deploy via Ansible em VPS BR genérica (fornecedor‑agnóstico).
  • Esteira: trunk‑based, tags duais — vX.Y.Z-rc.N dispara homologação automática, vX.Y.Z dispara produção com gate humano de 1 clique. Pre‑push hook local roda lint + análise estática + Pest + Dusk + cobertura antes de aceitar git push.
  • Observabilidade: logs JSON via Laravel, métricas persistidas em PostgreSQL, alertas via canal Telegram privado, mascaramento de PII em 3 camadas (sanitizer, exception em EventLogger, teste arquitetural).
  • Documentação viva: índice do projeto · relatório de validação do EPIC‑000 · status report interno · decisões arquiteturais (ADRs).
  • Repositório: github.com/xandroalmeida/defonline.
